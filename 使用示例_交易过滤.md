# 交易过滤功能使用示例

## 功能说明

交易过滤功能允许您只扫描指定的交易，而不是扫描jar包中的所有交易。这在以下场景中非常有用：

- 只需要验证特定交易的配置
- 增量更新某些交易的信息
- 调试特定交易的扫描问题

## 使用方法

### 1. 扫描所有交易（不过滤）

在"交易ID过滤"输入框中**留空**，系统会扫描jar包中的所有flowtrans.xml文件。

**示例：**
- Jar包路径：`D:\code\ccbs-online-dist.jar`
- 交易ID过滤：`（留空）`
- 选择jar包：`dept-pbf`

**结果：** 扫描dept-pbf中的所有交易

---

### 2. 扫描单个交易

在"交易ID过滤"输入框中输入一个交易ID。

**示例：**
- Jar包路径：`D:\code\ccbs-online-dist.jar`
- 交易ID过滤：`ba3401`
- 选择jar包：`ap-tran`

**结果：** 只扫描交易ID为`ba3401`的交易

---

### 3. 扫描多个交易

在"交易ID过滤"输入框中输入多个交易ID，用**英文逗号**分隔。

**示例：**
- Jar包路径：`D:\code\ccbs-online-dist.jar`
- 交易ID过滤：`ba3401,ba3402,dpcb2294`
- 选择jar包：`dept-pbf`

**结果：** 只扫描交易ID为`ba3401`、`ba3402`和`dpcb2294`的交易

---

### 4. 跨多个jar包扫描特定交易

可以同时选择多个jar包，系统会在所有选中的jar包中查找指定的交易。

**示例：**
- Jar包路径：`D:\code\ccbs-online-dist.jar`
- 交易ID过滤：`ba3401,dpcb2294,loan1001`
- 选择jar包：`ap-tran`, `dept-pbf`, `loan-pbf`（全选）

**结果：** 
- 在ap-tran中查找ba3401
- 在dept-pbf中查找dpcb2294
- 在loan-pbf中查找loan1001

---

## 注意事项

### 1. 交易ID的格式

- 交易ID对应flowtrans.xml文件中`<flowtran>`标签的`id`属性
- 例如：`<flowtran id="ba3401" longname="银承汇票签发取消">`
- 交易ID区分大小写

### 2. 分隔符

- 多个交易ID必须使用**英文逗号**（`,`）分隔
- ✅ 正确：`ba3401,ba3402,dpcb2294`
- ❌ 错误：`ba3401，ba3402，dpcb2294`（中文逗号）
- ❌ 错误：`ba3401 ba3402 dpcb2294`（空格）

### 3. 空格处理

系统会自动去除交易ID前后的空格，所以以下输入都是有效的：
- `ba3401,ba3402,dpcb2294`
- `ba3401, ba3402, dpcb2294`
- `ba3401 , ba3402 , dpcb2294`

### 4. 未找到的交易

如果指定的交易ID在jar包中不存在，不会报错，只是不会有数据入库。

查看扫描日志可以了解哪些交易被跳过：
```
交易ID ba9999 不在过滤列表中，跳过
```

## txnMode属性说明

### XML结构示例

```xml
<flowtran id="ba3401" longname="银承汇票签发取消" txnMode="A" package="cn.sunline.ltts.busi.lntran.trans.bkat">
    <!-- 交易配置 -->
</flowtran>
```

### txnMode取值规则

1. **有txnMode属性**：会保存到数据库的`txn_mode`字段
   ```xml
   <flowtran id="ba3401" txnMode="A">
   ```
   数据库：`txn_mode = 'A'`

2. **没有txnMode属性**：`txn_mode`字段为NULL
   ```xml
   <flowtran id="ba3402" longname="测试交易">
   ```
   数据库：`txn_mode = NULL`

3. **txnMode为空字符串**：`txn_mode`字段为NULL
   ```xml
   <flowtran id="ba3403" txnMode="">
   ```
   数据库：`txn_mode = NULL`

## 查询结果

扫描完成后，可以通过SQL查询结果：

### 查询所有扫描的交易

```sql
SELECT id, longname, txn_mode, from_jar, create_time 
FROM flowtran 
ORDER BY create_time DESC;
```

### 查询特定交易的流程步骤

```sql
SELECT fs.flow_id, fs.node_name, fs.node_type, fs.step, fs.node_longname
FROM flow_step fs
WHERE fs.flow_id = 'ba3401'
ORDER BY fs.step;
```

### 查询有txnMode的交易

```sql
SELECT id, longname, txn_mode, from_jar 
FROM flowtran 
WHERE txn_mode IS NOT NULL;
```

### 查询没有txnMode的交易

```sql
SELECT id, longname, from_jar 
FROM flowtran 
WHERE txn_mode IS NULL;
```

## 常见问题

### Q1：如何知道jar包中有哪些交易？

A：第一次可以不填写过滤条件，扫描全部交易，然后从数据库中查看：

```sql
SELECT id, longname FROM flowtran WHERE from_jar = 'dept-pbf';
```

### Q2：过滤条件会影响扫描速度吗？

A：会的。如果指定了交易ID过滤，系统会跳过不在过滤列表中的交易，从而加快扫描速度。

### Q3：可以使用通配符吗？

A：目前不支持通配符，必须输入完整的交易ID。

### Q4：大小写敏感吗？

A：是的，交易ID区分大小写。`ba3401`和`BA3401`是不同的交易。

## 提示

- 💡 留空交易ID过滤框，可以扫描所有交易
- 💡 使用逗号分隔多个交易ID
- 💡 系统会自动去除前后空格
- 💡 txnMode属性不存在时，数据库中该字段为NULL
