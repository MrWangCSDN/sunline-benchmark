# 应用启动说明

## 问题：APPLICATION FAILED TO START - DataSource 配置错误

### 原因
应用使用了 Spring Profile 来区分不同环境的配置（dev、sit），启动时需要激活对应的 profile。

### 解决方案

#### 方案 1：使用 IDE 启动（推荐）

在 IDE（如 IntelliJ IDEA）中配置启动参数：

1. **Edit Configurations**
2. 在 **VM options** 或 **Program arguments** 中添加：
   ```
   --spring.profiles.active=dev
   ```
   或在 **Environment variables** 中添加：
   ```
   SPRING_PROFILES_ACTIVE=dev
   ```

#### 方案 2：使用命令行启动

```bash
# Maven 启动
mvn spring-boot:run -Dspring-boot.run.profiles=dev

# 或者打包后启动
java -jar target/dict-manager.jar --spring.profiles.active=dev
```

#### 方案 3：使用环境变量

```bash
# Linux/Mac
export SPRING_PROFILES_ACTIVE=dev
java -jar target/dict-manager.jar

# Windows
set SPRING_PROFILES_ACTIVE=dev
java -jar target/dict-manager.jar
```

#### 方案 4：修改 application.yml（已完成）

确保 `application.yml` 中的配置正确：

```yaml
spring:
  application:
    name: dict-manager
  profiles:
    active: dev  # 默认激活 dev profile
```

**注意**：`profiles.active` 必须与 `application` 同级，不能有多余的空行。

## 可用的 Profile

- **dev**：开发环境（默认）
  - 数据库：103.47.81.50:3306/apimega
  
- **sit**：SIT 测试环境
  - 数据库：21.37.78.224:8890/twspdb

## 启动验证

启动成功后，应该能看到类似以下日志：

```
The following 1 profile is active: "dev"
...
Started DictManagerApplication in X.XXX seconds
```

## 如果仍然失败

1. **检查数据库连接**
   - 确保数据库服务器可访问
   - 确认用户名和密码正确
   - 检查防火墙设置

2. **检查 MySQL 驱动**
   - 确认 `pom.xml` 中包含 MySQL 驱动依赖

3. **查看完整错误日志**
   - 检查是否有其他配置错误

## 快速测试

如果只是想快速测试，可以临时修改 `application.yml`，将数据库配置移到主配置区域（不推荐生产使用）。
