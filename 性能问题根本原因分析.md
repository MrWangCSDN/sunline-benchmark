# Excel比对性能问题根本原因分析

## 🔴 核心问题：样式对象暴增导致性能退化

### 问题现象
```
表1比对：2秒
表10比对：3秒
表20比对：5秒
表30比对：10秒
表40比对：20秒  ← 越来越慢！
表50比对：40秒
表60比对：80秒  ← 指数级增长
```

### 根本原因

#### 1. POI样式对象限制
```
Excel文件最多支持：64,000个CellStyle对象
每创建一个CellStyle：
  - 占用内存：~1KB
  - 增加查找时间：O(n)
  - 触发GC：频繁分配对象
```

#### 2. 样式爆炸的代码模式
```java
// ❌ 错误示例（在循环中创建样式）
for (RevisionRecord record : revisionRecords) {  // 假设1000条
    CellStyle tableStyle = workbook.createCellStyle();  // 创建1000个
    CellStyle levelStyle = workbook.createCellStyle();  // 创建1000个
    CellStyle typeStyle = workbook.createCellStyle();   // 创建1000个
    CellStyle detailStyle = workbook.createCellStyle(); // 创建1000个
}
// 结果：4000个样式对象！
```

#### 3. 性能退化的数学模型
```
表数量 = n
平均每表行数 = m (假设100行)
每行单元格 = c (假设10列)

创建的样式数 = n × m × c × 4 (每个单元格可能4个操作)
            = n × 100 × 10 × 4
            = 4000n

样式查找时间 = O(样式数²) = O(n²)
```

**为什么越来越慢？**
- 10个表：40,000个样式，查找时间 O(10²)
- 50个表：200,000个样式（超限！），查找时间 O(50²) = **2500倍慢！**

### 修复方案

#### ✅ 方案1：样式复用（已实施）
```java
// 只创建4个样式，全局复用
private static class StyleCache {
    CellStyle modifiedBg;  // 1个
    CellStyle addedBg;     // 1个
    CellStyle deletedBg;   // 1个
    CellStyle normal;      // 1个
}

// 结果：总共4个样式对象，无论多少表！
```

#### ✅ 方案2：禁用样式复制（已实施）
```java
// 不复制原始Excel的样式，只复制值
copyCellValueOnly(sourceCell, targetCell);

// 而不是：
// copyCell(sourceCell, targetCell);  // 会创建新样式
```

#### ✅ 方案3：简化标记逻辑（已实施）
```java
// 修订记录sheet中，不为每个单元格创建样式
// 只在需要时创建最少量的样式
```
