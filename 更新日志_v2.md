# 更新日志 - v2.0

## 更新时间
2025年12月

## 主要更新

### 1. ✅ 修复文件名匹配错误

**问题：** 代码中查找`.flowtran.xml`，但实际文件名是`.flowtrans.xml`（多了一个`s`）

**修复：**
- 将所有文件匹配逻辑从`.flowtran.xml`改为`.flowtrans.xml`
- 更新相关注释和文档

**影响：** 修复后可以正确识别和扫描jar包中的交易定义文件

---

### 2. ✨ 新增交易过滤功能

**功能描述：**
在扫描页面增加"交易ID过滤"输入框，支持指定要扫描的交易。

**使用方法：**
- 输入单个交易ID：`ba3401`
- 输入多个交易ID：`ba3401,ba3402,dpcb2294`（用英文逗号分隔）
- 留空：扫描所有交易（默认行为）

**实现细节：**
- 前端：新增交易过滤输入框
- 后端：
  - Service接口新增`transactionIds`参数
  - 解析XML时检查交易ID是否在过滤列表中
  - 不在列表中的交易直接跳过，不解析也不入库

**优势：**
- 提高扫描效率（只扫描需要的交易）
- 支持增量更新
- 便于问题排查和调试

---

### 3. 🔧 修正txnMode字段取值来源

**问题：** 之前从`kind`属性读取，但应该从`txnMode`属性读取

**修复前：**
```xml
<flowtran id="ba3401" kind="3" txnMode="A">
```
读取：`txn_mode = "3"` ❌

**修复后：**
```xml
<flowtran id="ba3401" kind="3" txnMode="A">
```
读取：`txn_mode = "A"` ✅

**特殊处理：**
- 如果XML中没有`txnMode`属性，则`txn_mode`字段不落库（为NULL）
- 如果`txnMode`为空字符串，也不落库
